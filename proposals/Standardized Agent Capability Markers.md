# Standardized Agent Capability Markers

## Purpose
Implement standardized capability markers in documentation to clearly indicate which AI agents can effectively work with specific documents or features.

## Key Components
1. Capability Tags System - Standardized tags at the top of documents
2. Core Capability Types - LONG_CONTEXT, CODE_EXECUTION, TECHNICAL_REASONING, MEMORY_ACCESS, MULTI_AGENT
3. Implementation - Add markers to templates, update existing docs, include in PR template

## Benefits
- Prevents workflow disruptions when documents are assigned to incompatible agents
- Improves collaboration by setting clear expectations
- Simplifies task routing in the SkogAI ecosystem
- Reduces redundant capability checking
